
EXPERIMENT 6: IMPLEMENTATION OF PROJECT AND INTEGRATION OF MODULES

AIM:
To implement the BookStore-CMS project with proper module organization and verify
successful integration of all modules (Frontend, Backend, Database, Cache, Search).

PROJECT OVERVIEW:
BookStore-CMS is a full-stack, production-grade bookstore management system
implementing multi-tier architecture with:
- Frontend: React 18 + Vite + TypeScript
- Backend: NestJS + TypeScript
- Database: PostgreSQL + Prisma ORM
- Cache: Redis
- Search: Meilisearch
- Container: Docker + Docker Compose

ARCHITECTURE & MODULE STRUCTURE: 

1. FRONTEND MODULE (React + Vite)
   Components:
   - Authentication Pages (Login, SignUp, ForgotPassword)
   - Admin Dashboard & Management Pages
   - Customer Dashboard & Shopping Pages
   - Shared UI Components Library
   - API Client & HTTP interceptors
   
   Key Features:
   - Single Page Application (SPA)
   - React Query for server state
   - Type-safe API client
   - Responsive design with TailwindCSS

2. BACKEND MODULE (NestJS API)
   Modules:
   - Auth Module: JWT authentication, token refresh, password reset
   - Inventory Module: Book CRUD, Categories, Authors, Publishers
   - Orders Module: Order management, Returns, Coupons, Invoicing
   - Cart Module: Shopping cart operations, Checkout
   - Customers Module: Customer profiles, Loyalty programs
   - Suppliers Module: Supplier management, Purchase orders
   - Analytics Module: Sales reports, Revenue analysis
   - Notifications Module: Event-driven notification system
   - Books Reading Module: Digital book management
   - API Compatibility Module: Unified endpoint layer

   Architecture Layers:
   - Controllers: HTTP request handling
   - Services: Business logic implementation
   - Guards: Authentication & Authorization
   - Interceptors: Request/Response transformation
   - Filters: Exception handling
   - Middlewares: Logging, rate limiting
   - Database Layer: Prisma ORM integration

3. DATABASE MODULE (PostgreSQL + Prisma)
   Schema Components:
   - 30+ Data Models organized in entity groups
   - User & Authentication Models
   - Inventory Models (Books, Authors, Publishers, Categories)
   - Order & Transaction Models
   - Customer & Loyalty Models
   - Supplier & Purchase Order Models
   - Analytics & Audit Models
   - Relationships: 50+ Foreign keys with constraints
   - Migrations: Version-controlled schema changes
   - Indexes: Performance optimization
   
4. SEARCH ENGINE (Meilisearch)
   Indexing:
   - Book catalog full-text search
   - Faceted search by category, author, price
   - Instant search results

5. INFRASTRUCTURE MODULE (Docker )
   Services:
   - PostgreSQL Database Container
   - Meilisearch Search Container


MODULE INTEGRATION POINTS:

1. FRONTEND ↔ BACKEND INTEGRATION
   - HTTP/HTTPS communication using Axios
   - JSON data format
   - JWT token in Authorization headers
   - Centralized API client (api-client.ts)
   - Error handling & retry logic
   - Request/Response interceptors
   - Type-safe endpoint definitions

2. BACKEND ↔ DATABASE INTEGRATION
   - Prisma ORM abstracts PostgreSQL queries
   - Type-safe database operations
   - Automatic migration management
   - Connection pooling
   - Transaction support for complex operations
   - Relationship lazy loading

3. BACKEND ↔ MEILISEARCH INTEGRATION
   - Index synchronization on book create/update
   - Search API endpoints
   - Faceted filtering support

4. MULTIPLE MODULES COORDINATION
   - Order → Inventory coordination (stock deduction)
   - Order → Notifications coordination (event publishing)
   - Cart → Inventory coordination (availability check)
   - Auth → All modules (RBAC enforcement)
   - Analytics → All modules (data aggregation)
