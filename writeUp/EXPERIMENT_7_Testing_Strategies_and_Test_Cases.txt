# EXPERIMENT 7: DEVELOPMENT OF TESTING STRATEGIES AND TEST CASES

## AIM

To develop testing strategies and test cases for the BookStore-CMS project using Basis Path Testing and Cyclomatic Complexity analysis.

## TESTING STRATEGY

### Basis Path Testing

Basis Path Testing is a white-box testing technique that analyzes the control flow of a program and identifies independent execution paths. The number of required test cases is determined using Cyclomatic Complexity.

**Cyclomatic Complexity Formula:**

M = E - N + 2P

Where:

* E = Number of edges
* N = Number of nodes
* P = Number of connected components

### Testing Process

1. Identify decision points in the code.
2. Calculate cyclomatic complexity.
3. Determine independent execution paths.
4. Design test cases for each path.
5. Verify path and decision coverage.

## CONCLUSION

Testing strategies and test cases were successfully developed for the BookStore-CMS project using the Basis Path Testing approach. Cyclomatic Complexity was calculated for critical modules to identify independent execution paths and determine the minimum number of required test cases. The developed test cases cover major business functionalities including authentication, inventory management, order processing, notifications, and analytics. The analysis shows that the system has moderate complexity with an average cyclomatic complexity of 5.1, indicating good maintainability and manageable control flow. The testing strategy provides systematic coverage of important execution paths and helps improve software quality, reliability, and defect detection efficiency.
