INTEGRATION TEST FRAMEWORKS
================================================================================

- Framework: Jest + Supertest + NestJS Testing Module

- Approach: Database transactions with rollback
- Fixtures: Seed data for consistent state
- Coverage: Basis path testing for multi-module interactions

Frontend Integration Tests:
- Framework: Vitest + React Testing Library
- Approach: Mock API responses with MSW
- Testing: Component interactions and data flows
- Validation: User interactions and page state
- Coverage: Control flow through UI components

API Integration Tests:
- Framework: Supertest + Jest
- Approach: Start real server, send HTTP requests
- Validation: HTTP status codes, response bodies
- Authentication: Include JWT tokens in requests
- Coverage: Decision points in endpoint handlers

E2E Tests:
- Framework: Playwright
- Approach: Real browser automation
- Environment: Containerized services
- Coverage: Critical user paths with complexity analysis

TEST CASE 9.1: ORDER WORKFLOW - AUTHENTICATION TO CHECKOUT (M=7 PATHS)
Objective: Test all 7 basis paths through complete order creation flow
Test Type: Integration Test (Multi-Module)
Modules Tested: Auth → Cart → Inventory → Payment → Orders → Notifications
Cyclomatic Complexity: M = 7
Coverage Target: 100% of paths

Basis Path Test Suite (7 Required Tests):

[PATH 1] - Unauthenticated Access Decision Branch
  Precondition: No JWT token provided
  Workflow: Customer clicks checkout without login
  Execution: Frontend sends request → API verifies token → INVALID
  Expected: 401 Unauthorized, redirect to login page
  Assertion: expect(response.status).toBe(401); expect(response.body.message).toContain('Unauthorized')
  Coverage: Authentication middleware decision point

[PATH 2] - Invalid Cart Validation Decision Branch
  Precondition: Authenticated user, empty cart
  Workflow: User attempts checkout with empty cart
  Execution: Cart service validates items → EMPTY
  Expected: 400 Bad Request
  Assertion: expect(response.status).toBe(400); expect(response.body.error).toContain('Cart empty')
  Coverage: Cart validation decision

[PATH 3] - Inventory Insufficient Decision Branch
  Precondition: Authenticated, cart valid, book out of stock
  Workflow: Order processing attempts inventory reservation → Stock insufficient
  Execution: Inventory service checks stock → NOT_AVAILABLE
  Expected: 409 Conflict
  Assertion: expect(response.status).toBe(409); expect(response.body.unavailableItems.length).toBeGreaterThan(0)
  Coverage: Inventory check decision

[PATH 4] - Invalid Coupon Decision Branch
  Precondition: Authenticated, cart valid, stock available, invalid coupon applied
  Workflow: Order processing validates coupon → Invalid format
  Execution: Coupon service validates → INVALID
  Expected: 200 OK, order continues without discount
  Assertion: expect(response.status).toBe(200); expect(response.body.couponApplied).toBeFalsy()
  Coverage: Coupon validation graceful failure

[PATH 5] - Payment Gateway Timeout Decision Branch
  Precondition: All valid, payment processing timeout
  Workflow: Payment processor delays beyond timeout threshold
  Execution: Payment service calls gateway → TIMEOUT
  Expected: 503 Temporary failure, retry queued
  Assertion: expect(response.status).toBe(503); expect(mockRetryQueue.add).toHaveBeenCalled()
  Coverage: Payment timeout handling

[PATH 6] - Notification Service Failure Decision Branch
  Precondition: All valid, order created, notification service fails
  Workflow: Order created successfully, notification send fails
  Execution: Order created → Notification queued → Service error
  Expected: 200 OK (order confirmed despite notification failure)
  Assertion: expect(response.status).toBe(200); expect(response.body.orderId).toBeDefined(); expect(mockNotificationQueue.enqueue).toHaveBeenCalled()
  Coverage: Failure resilience decision

[PATH 7] - Complete Success Decision Branch (Happy Path)
  Precondition: All systems operational
  Workflow: User authenticated → Cart valid → Stock available → Coupon valid → Payment OK → Notification sent
  Execution: Complete order flow → All checks → SUCCESS
  Expected: 201 Created, order object returned
  Assertion: expect(response.status).toBe(201); expect(response.body.confirmationNumber).toBeDefined(); expect(response.body.estimatedDelivery).toBeDefined()
  Coverage: All decision points pass

Integration Coverage: 100% (7/7 paths)
Cyclomatic Complexity: M = 7
Coverage Achieved: 100% basis path coverage

Execution:
  - Run all integration tests: `pnpm run test:integration`
  - Run specific test: `pnpm run test:integration -- --testNamePattern="ORDER WORKFLOW"`
  - With coverage: `pnpm run test:integration:cov`
