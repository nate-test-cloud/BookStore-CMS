EXPERIMENT 7: TESTING STRATEGIES USING BASIS PATH TESTING METHODOLOGY
================================================================================

OBJECTIVE: Develop testing strategy using basis path testing (McCabe's approach)

METHODOLOGY: Basis Path Testing (Cyclomatic Complexity Analysis)
- Formula: M = E - N + 2P (Edges - Nodes + 2×Connected Components)
- Coverage: 100% of all independent paths
- Quality Threshold: M ≤ 7 (ideal average M = 5.1)

TESTING STRATEGY FRAMEWORK:

1. BASIS PATH TESTING APPROACH
   - Analyzes control flow graph of each method
   - Calculates cyclomatic complexity (M)
   - Derives M linearly independent paths
   - Creates minimum M test cases per method
   - Achieves 100% decision and path coverage

   Advantages:
   ✓ Systematic and quantifiable
   ✓ Identifies all execution paths
   ✓ Catches edge cases and boundary conditions
   ✓ Measurable coverage metrics (independent paths)
   ✓ Reduces defect escape rate significantly

CRITICAL WORKFLOW ANALYSIS:

ORDER CREATION WORKFLOW (M=7 Independent Paths)
Control Flow:
  [P1] Unauthenticated → 401 Error
  [P2] Empty cart → 400 Error
  [P3] Stock unavailable → 409 Error
  [P4] Invalid coupon → Continue without discount
  [P5] Payment fails → Rollback
  [P6] Notification fails → Retry
  [P7] All pass → Order created

CYCLOMATIC COMPLEXITY BY MODULE:
  - Auth Service (signup): M = 5
  - Auth Service (login): M = 4
  - Inventory Service (adjustStock): M = 5
  - Cart Service (checkout): M = 4
  - Orders Service (createOrder): M = 7
  - Notifications Service (publish): M = 4
  - Suppliers Service (createPO): M = 5
  - Analytics Service (generateReport): M = 6
  - AVERAGE: M = 5.1 (Excellent range)
